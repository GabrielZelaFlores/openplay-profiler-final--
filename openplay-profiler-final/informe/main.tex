\begin{abstract}
Este informe presenta \textit{OpenPlay Profiler}, una herramienta de ciencia de datos y visualizaci\'on anal\'itica orientada a explorar el dataset OpenPlay. La propuesta integra variables demogr\'aficas, bienestar, riesgo de juego problem\'atico, telemetr\'ia, cognici\'on y uso del tiempo para construir un vector de caracter\'isticas por participante. A partir de ese vector se aplican t\'ecnicas de reducci\'on dimensional, clustering y vistas coordinadas que permiten identificar similitudes, diferencias, anomal\'ias y perfiles exploratorios. El aporte central del trabajo no es solo generar gr\'aficos, sino validar que la herramienta permite extraer conocimiento mediante estudios de caso sustentados en m\'as de una vista visual.
\end{abstract}

\noindent\textbf{Palabras clave:} OpenPlay, videojuegos, bienestar digital, visualizaci\'on anal\'itica, PCA, UMAP, t-SNE, clustering, perfiles exploratorios.

\section{Introducci\'on}

Los videojuegos son una actividad digital masiva y generan datos sobre frecuencia de uso, plataformas, sesiones, horarios y respuestas de los usuarios. En este contexto, el estudio OpenPlay recopila telemetr\'ia de juego de plataformas como Nintendo Switch, Steam y Xbox, junto con medidas psicol\'ogicas y de bienestar como WEMWBS, PROMIS, GDT y BANGS, adem\'as de variables de motivaci\'on y cognici\'on \cite{openplay}. Adem\'as, la Organizaci\'on Mundial de la Salud define el trastorno por videojuegos como un patr\'on de comportamiento caracterizado por control deteriorado, prioridad creciente del juego y continuidad pese a consecuencias negativas \cite{who}. Por ello, analizar datos de juego y bienestar es un problema relevante tanto a nivel social como computacional.

El problema de ciencia de datos abordado consiste en identificar perfiles de jugadores dentro de un conjunto de datos multidimensional. En este informe, los perfiles exploratorios son agrupamientos preliminares de participantes con valores similares en las variables analizadas; se denominan exploratorios porque sirven para describir patrones y generar hip\'otesis, no para establecer diagn\'osticos ni categor\'ias definitivas. Espec\'ificamente, se busca reconocer qu\'e participantes tienen comportamientos similares y qu\'e variables explican la separaci\'on entre grupos. Una anomal\'ia se entiende como un participante o subgrupo con valores alejados del patr\'on general, por ejemplo, sesiones nocturnas muy superiores al promedio o baja cobertura de registros frente al resto de la muestra.

Para abordar el problema se construye un vector de caracter\'isticas por participante. Este vector integra variables demogr\'aficas, bienestar, riesgo de juego problem\'atico, telemetr\'ia, actividad diaria, cognici\'on y uso del tiempo. La selecci\'on de estas dimensiones responde a la estructura del dataset OpenPlay y permite comparar participantes usando una representaci\'on com\'un. Antes del agrupamiento, las variables se revisan y estandarizan para reducir el efecto de escalas diferentes y facilitar la comparaci\'on entre atributos.

El objetivo del proyecto es identificar y caracterizar perfiles exploratorios de jugadores en OpenPlay mediante reducci\'on dimensional y clustering. La herramienta \textit{OpenPlay Profiler} se usa como apoyo para ejecutar el flujo de an\'alisis, observar la proyecci\'on bidimensional y comparar los grupos encontrados. Los resultados no se presentan como diagn\'osticos cl\'inicos, sino como agrupamientos exploratorios que pueden servir a investigadores, analistas de comportamiento y dise\~nadores de experiencias digitales para formular hip\'otesis sobre patrones de juego y bienestar.

\section{Trabajos relacionados}

Los trabajos relacionados se agrupan en dos l\'ineas: estudios sobre datos de videojuegos y bienestar, y propuestas de visualizaci\'on anal\'itica para datos multivariados. Esta revisi\'on no busca cubrir toda la literatura, sino ubicar el proyecto dentro de enfoques cercanos y justificar por qu\'e una herramienta visual interactiva es pertinente para el problema.

\subsection{Trabajos relacionados con modelos y m\'etodos}

El estudio OpenPlay plantea una base de datos longitudinal que combina trazas digitales de juego con medidas psicol\'ogicas y cognitivas \cite{openplay}. Su importancia para este proyecto est\'a en que no limita el an\'alisis a una sola fuente: incluye plataformas, encuestas, tareas cognitivas y uso del tiempo. Esa estructura exige t\'ecnicas capaces de integrar variables heterog\'eneas y revisar la cobertura de cada fuente antes de interpretar resultados.

La Organizaci\'on Mundial de la Salud define el trastorno por videojuegos en la CIE-11 como un patr\'on persistente de comportamiento de juego con deterioro del control, prioridad creciente y continuidad pese a consecuencias negativas \cite{who}. Esta referencia no se usa para diagnosticar participantes, sino para justificar que el an\'alisis de patrones de juego y bienestar tiene relevancia social y debe tratarse con cuidado metodol\'ogico.

En cuanto a m\'etodos, PCA, UMAP y t-SNE son t\'ecnicas utilizadas para representar datos de muchas dimensiones en espacios de menor dimensi\'on \cite{jolliffe,umap,tsne}. Su funci\'on en este proyecto es permitir una lectura visual de similitudes entre participantes. El clustering, especialmente K-means, complementa esa proyecci\'on al separar grupos preliminares que luego deben interpretarse con variables originales \cite{kmeans}.

\subsection{Trabajos relacionados con visualizaci\'on anal\'itica}

La visualizaci\'on anal\'itica combina representaciones visuales, interacci\'on humana y m\'etodos computacionales para apoyar el razonamiento sobre datos complejos \cite{keim}. Heer y Shneiderman destacan que la interacci\'on es esencial para explorar, filtrar, seleccionar y conectar vistas durante el proceso de an\'alisis \cite{heer}. Este enfoque se relaciona directamente con OpenPlay Profiler, porque el usuario no solo observa una proyecci\'on 2D: tambi\'en selecciona participantes, revisa variables originales, aplica filtros, compara grupos y valida si un patr\'on aparece en varias vistas.

\section{Modelos}

El proyecto no utiliza modelos predictivos ni clasificadores supervisados, porque no existe una etiqueta cl\'inica que se busque predecir. El objetivo es exploratorio: construir una representaci\'on com\'un de cada participante, proyectarla visualmente, detectar perfiles y explicar esos perfiles con variables originales. Por esa raz\'on, los modelos se usan como apoyo para la visualizaci\'on anal\'itica y no como mecanismos autom\'aticos de diagn\'ostico.

En esta secci\'on se describen los modelos como componentes del sistema: qu\'e reciben, qu\'e transforman y qu\'e entregan a las vistas. Las im\'agenes de resultados se reservan para la secci\'on de validaci\'on, donde se muestran como evidencia de los casos de estudio.

\subsection{Vector de caracter\'isticas}

El vector de caracter\'isticas es la representaci\'on num\'erica de cada participante. En este proyecto integra variables de varias dimensiones: edad y plataformas, escalas de bienestar y riesgo, sesiones de juego, actividad nocturna, juegos \'unicos, registros diarios, medidas cognitivas y uso del tiempo. Su utilidad es convertir datos de origen distinto en una matriz comparable para los modelos.

El vector se conecta con las visualizaciones porque determina la posici\'on de cada participante en PCA, UMAP o t-SNE. Si el vector se modifica, tambi\'en cambia la proyecci\'on y la lectura de clusters. Por esa raz\'on la herramienta incluye una vista de evaluaci\'on del vector, donde se revisa si las variables elegidas son num\'ericas, si tienen cobertura suficiente y si existen correlaciones altas que puedan generar redundancia. En los casos de estudio, el vector permite combinar telemetr\'ia, bienestar y uso del tiempo en una misma lectura visual.

Este vector funciona como configuraci\'on inicial para la exploraci\'on, aunque la interfaz permite agregar o quitar variables antes de ejecutar reducci\'on dimensional o clustering.

\subsection{Estandarizaci\'on e imputaci\'on}

Antes de aplicar reducci\'on dimensional se estandarizan las variables num\'ericas. Esto evita que variables con escalas grandes, como minutos de juego o tiempos de respuesta, dominen sobre variables con escalas peque\~nas, como puntajes de encuestas. Para los valores faltantes, la herramienta permite imputaci\'on por media, imputaci\'on con cero o eliminaci\'on de casos, de modo que el usuario pueda evaluar c\'omo cambia la proyecci\'on seg\'un el tratamiento elegido.

En la herramienta, la estandarizaci\'on y el tratamiento de valores faltantes se aplican antes de la reducci\'on dimensional y el clustering. Esta decisi\'on permite comparar variables de escalas distintas y evita que las sesiones o minutos de juego oculten las diferencias en escalas como PROMIS, WEMWBS, GDT o BANGS.

\subsection{Correlaci\'on}

La correlaci\'on se usa para revisar relaciones entre variables num\'ericas y detectar redundancia. En la herramienta, la matriz de correlaci\'on no elimina autom\'aticamente variables, pero advierte pares con relaci\'on alta. Esto ayuda a decidir si el vector est\'a demasiado concentrado en una misma familia de medidas o si representa dimensiones distintas del problema.

La correlaci\'on tambi\'en cumple una funci\'on de validaci\'on: cuando una separaci\'on visual se plantea como relaci\'on entre dos variables, la vista bivariada y la matriz de correlaci\'on permiten revisar si esa relaci\'on tiene respaldo en los valores originales. Esto es relevante para el Caso 2, donde las sesiones totales y nocturnas se analizan juntas.

\subsection{PCA}

PCA, o an\'alisis de componentes principales, transforma un conjunto de variables correlacionadas en componentes lineales que concentran la mayor variabilidad posible \cite{jolliffe}. En el sistema se utiliza para proyectar participantes en dos ejes interpretables como componentes principales. El resultado es un mapa donde la cercan\'ia entre puntos indica similitud aproximada seg\'un el vector seleccionado. La herramienta tambi\'en reporta la varianza explicada, lo que permite saber si la vista 2D conserva suficiente informaci\'on para apoyar una lectura exploratoria.

En la validaci\'on del informe, PCA se usa como mapa de referencia para ubicar los perfiles C0, C1, C2 y C3. Sin embargo, la conclusi\'on no depende solo de la ubicaci\'on de los puntos: cada grupo se explica despu\'es mediante medias estandarizadas, cobertura por fuente y relaciones bivariadas.

\subsection{UMAP}

UMAP construye una representaci\'on de baja dimensi\'on conservando relaciones de vecindad entre observaciones \cite{umap}. En este proyecto se usa para observar grupos locales de participantes que tienen combinaciones similares de bienestar, telemetr\'ia, cognici\'on y uso del tiempo. Sus par\'ametros, como n\'umero de vecinos y distancia m\'inima, permiten modificar el nivel de separaci\'on visual. El conocimiento que aporta es la identificaci\'on de zonas de similitud, separaciones entre perfiles y posibles puntos at\'ipicos.

UMAP es \'util durante la exploraci\'on interactiva porque puede separar vecindarios locales que PCA no siempre separa. Aun as\'i, cualquier patr\'on encontrado con UMAP debe confirmarse con variables originales, filtros y vistas bivariadas para evitar interpretar una forma visual como evidencia suficiente.

\subsection{t-SNE}

t-SNE es una t\'ecnica de reducci\'on dimensional que busca preservar vecindades locales en una proyecci\'on de baja dimensi\'on \cite{tsne}. En la herramienta funciona como alternativa a PCA y UMAP. Su valor est\'a en comparar si los grupos locales se mantienen cuando se usa otro m\'etodo de proyecci\'on. Si un patr\'on aparece en una proyecci\'on y tambi\'en se revisa con variables originales o con otra vista, la interpretaci\'on gana solidez.

\subsection{Clustering}

El clustering agrupa participantes seg\'un su similitud. OpenPlay Profiler incluye K-means, clustering jer\'arquico de enlace simple y DBSCAN. K-means asigna cada punto al centroide m\'as cercano y recalcula centros hasta estabilizar los grupos \cite{kmeans}. DBSCAN busca regiones densas y permite separar ruido, mientras que el m\'etodo jer\'arquico permite observar agrupamientos por distancia. El resultado de estos modelos es una etiqueta de grupo para cada participante y un conjunto de m\'etricas descriptivas, como silhouette aproximado, cohesi\'on, separaci\'on y balance de tama\~no. Estas etiquetas se conectan con la vista de proyecci\'on 2D y con la interpretaci\'on por variables originales.

En la validaci\'on se usa clustering para organizar participantes en perfiles exploratorios. El cluster no se interpreta por su color, sino por las variables que lo diferencian: cobertura multiplataforma, intensidad nocturna, bienestar/riesgo y uso registrado. De esta forma, el modelo cumple una funci\'on de descubrimiento visual, mientras que la explicaci\'on se sostiene en las vistas de confirmaci\'on.

\section{Descripci\'on general del sistema / Overview}

OpenPlay Profiler est\'a organizado como una aplicaci\'on web interactiva para convertir datos heterog\'eneos de juego y bienestar en evidencia visual verificable. El usuario carga el dataset consolidado o archivos fuente, revisa la calidad de los datos, selecciona variables, eval\'ua el vector, aplica reducci\'on dimensional, ejecuta clustering y valida patrones mediante vistas conectadas. La propuesta no termina en la generaci\'on de una proyecci\'on 2D: el flujo obliga a revisar cobertura, variables originales y casos de estudio antes de formular conclusiones.

\subsection{Descripci\'on de los datos}

La fuente principal es el dataset OpenPlay, consolidado en el archivo \texttt{openplay\_consolidated.csv}. En la versi\'on utilizada por la herramienta, el archivo contiene 2544 participantes y 186 variables. De ellas, 132 se tratan como num\'ericas y 54 como categ\'oricas dentro del sistema. El porcentaje promedio de valores faltantes calculado sobre las columnas consolidadas es 52.5\%. Este dato no se interpreta como resultado del comportamiento de los jugadores, sino como una condici\'on de calidad que afecta la lectura visual: antes de explicar perfiles, el usuario debe saber qu\'e fuentes tienen suficiente cobertura.

Las fuentes integradas incluyen encuestas de ingreso, encuestas biweekly, encuestas diarias, telemetr\'ia de Steam, Xbox, Nintendo Switch, Android e iOS, tareas cognitivas y registros de uso del tiempo. Las variables principales se agrupan de la siguiente manera:

\begin{itemize}
    \item \textbf{Demograf\'ia y base del participante:} edad, g\'enero, pa\'is, educaci\'on, plataformas declaradas y cobertura de telemetr\'ia.
    \item \textbf{Bienestar y salud mental:} puntajes como WEMWBS y PROMIS.
    \item \textbf{Riesgo de juego problem\'atico:} GDT y BANGS.
    \item \textbf{Telemetr\'ia de juego:} sesiones totales, sesiones nocturnas, duraci\'on de sesiones, juegos \'unicos y tiempo de juego en Steam.
    \item \textbf{Actividad diaria:} d\'ias con registros, d\'ias jugados y reportes asociados a la actividad.
    \item \textbf{Cognici\'on:} tiempo de respuesta promedio, precisi\'on promedio, n\'umero de tareas y ensayos.
    \item \textbf{Uso del tiempo:} n\'umero de entradas, d\'ias registrados y entradas relacionadas con videojuegos.
\end{itemize}

El dataset es multivariado, heterog\'eneo y parcialmente temporal. Contiene variables num\'ericas, categ\'oricas y banderas de disponibilidad. Tambi\'en posee una dimensi\'on temporal indirecta porque varias variables resumen sesiones, d\'ias registrados o actividad durante ventanas de tiempo. En el sistema, las banderas de disponibilidad son importantes porque permiten diferenciar patrones reales de comportamiento frente a patrones producidos por baja observabilidad.

\subsection{Preprocesamiento}

El preprocesamiento se concentra en preparar datos para an\'alisis visual, no en producir un an\'alisis estad\'istico cerrado. Primero se integran las fuentes por identificador de participante. Luego se generan variables derivadas, como sesiones totales, sesiones nocturnas, juegos \'unicos, minutos de juego, conteos de registros y banderas de disponibilidad. Despu\'es se revisan tipos de datos, valores faltantes, distribuciones y cobertura. Esta etapa responde directamente al Caso 1, porque la cobertura de datos condiciona la fuerza de cualquier interpretaci\'on posterior.

Para la reducci\'on dimensional, solo se usan variables num\'ericas. Las variables categ\'oricas se conservan para filtrar, describir e interpretar participantes, pero no entran directamente al c\'alculo de PCA, UMAP o t-SNE. Las variables seleccionadas se estandarizan y se tratan los valores faltantes seg\'un el m\'etodo elegido. Finalmente, las coordenadas 2D y las etiquetas de clustering se conectan con las vistas de selecci\'on, tablas y perfiles. La interpretaci\'on final no se basa solamente en la posici\'on de los puntos, sino en la combinaci\'on entre proyecci\'on, diferencias estandarizadas, filtros y gr\'aficos de validaci\'on.

\subsection{Preguntas gu\'ia del an\'alisis}

Las tareas del sistema se formularon a partir de preguntas concretas que s\'i pueden responderse con las vistas implementadas en OpenPlay Profiler. De esta forma, ninguna tarea queda desconectada de la interfaz ni de los datos reales del proyecto.

\begin{itemize}
    \item \textbf{P1.} ?Qu\'e fuentes de OpenPlay tienen suficiente cobertura para sostener una interpretaci\'on visual y cu\'ales deben leerse con cautela?
    \item \textbf{P2.} ?Qu\'e relaciones entre telemetr\'ia, bienestar, riesgo y uso del tiempo explican diferencias entre participantes?
    \item \textbf{P3.} ?Qu\'e perfiles exploratorios aparecen al combinar el vector de caracter\'isticas con reducci\'on dimensional y clustering, y c\'omo se validan con variables originales?
\end{itemize}

\section{Dise\~no visual}

La interfaz se organiza en pesta\~nas: Datos, Profiling, Bivariado, Encuestas, Vector, Filtros, PCA/t-SNE/UMAP y Validaci\'on. Esta organizaci\'on sigue el flujo del an\'alisis: primero se entiende el dataset, luego se revisan variables, se construye el vector, se aplican filtros, se proyectan participantes, se interpretan perfiles y finalmente se validan los patrones mediante estudios de caso.

El dise\~no visual no fue planteado como una colecci\'on de gr\'aficos independientes. Cada vista cumple una funci\'on dentro del razonamiento anal\'itico. La vista de datos permite decidir si una fuente es confiable; profiling ayuda a revisar calidad de variables; bivariado permite comprobar relaciones espec\'ificas; encuestas separa las escalas psicol\'ogicas; vector justifica qu\'e atributos entran al modelo; reducci\'on dimensional permite observar perfiles; filtros comprueban si un patr\'on se mantiene en una submuestra; y validaci\'on resume los casos de estudio con evidencia visual.

\subsection{Coordinaci\'on entre vistas}

La coordinaci\'on es importante porque evita interpretar un patr\'on desde una sola imagen. Por ejemplo, un grupo separado en la proyecci\'on puede revisarse con la tabla de variables originales; si adem\'as se filtra por cobertura y se compara en la vista bivariada, el usuario puede distinguir entre un perfil real y un efecto producido por datos faltantes. Del mismo modo, un perfil de juego nocturno o un perfil multiplataforma puede contrastarse con las variables originales, las banderas de disponibilidad y los filtros. Esta relaci\'on entre vistas convierte el sistema en una herramienta de an\'alisis visual y no solo en un conjunto de gr\'aficos independientes.

El flujo de validaci\'on funciona de la siguiente manera. Primero, el usuario identifica una posible separaci\'on o concentraci\'on en la vista de reducci\'on dimensional. Luego revisa qu\'e variables diferencian al grupo en la tabla de interpretaci\'on. Despu\'es usa una vista espec\'ifica, como bivariado, encuestas, cobertura o filtros, para confirmar si el patr\'on se mantiene. Finalmente, la vista de validaci\'on resume el caso con una figura y una lectura concreta. Este flujo responde directamente a la indicaci\'on de validar conocimiento mediante varias vistas conectadas.

\subsection{Caso de estudio 1: cobertura de datos y confiabilidad del an\'alisis}

\textbf{Pregunta anal\'itica.} ?Qu\'e fuentes de OpenPlay tienen suficiente cobertura para sostener una interpretaci\'on visual y cu\'ales deben leerse con mayor cautela?

\textbf{Vistas utilizadas.} Vista de datos, resumen BI, profiling, filtros por banderas de disponibilidad y vista de vector de caracter\'isticas.

\textbf{Variables analizadas.} \texttt{has\_steam}, \texttt{has\_xbox}, \texttt{has\_nintendo}, \texttt{has\_android}, \texttt{has\_ios}, \texttt{has\_cognitive}, \texttt{has\_timeuse}, porcentaje de faltantes y tipo de variable.

\textbf{Interacci\'on.} El usuario revisa la cobertura general en la vista de datos, selecciona columnas en profiling para observar faltantes y tipos de dato, y usa filtros por banderas de disponibilidad para comprobar cu\'antos participantes quedan cuando se exige una fuente espec\'ifica.

\textbf{Patr\'on encontrado.} La cobertura no es uniforme entre fuentes. Steam tiene una presencia mucho mayor que Android, iOS o Xbox, mientras que cognici\'on y uso del tiempo tienen cobertura intermedia. Este patr\'on indica que las conclusiones sobre actividad en Steam pueden apoyarse en m\'as casos, pero las lecturas sobre m\'oviles o Xbox deben tratarse como submuestras con menor representaci\'on.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.92\textwidth]{figures/fig01_cobertura_fuentes.png}
    \caption{Caso 1: cobertura por fuente integrada en OpenPlay. Steam, biweekly y cognici\'on tienen mayor presencia, mientras que Android, iOS y Xbox aparecen como fuentes de baja cobertura.}
    \label{fig:cobertura_fuentes}
\end{figure}

\textbf{Validaci\'on cruzada.} La Figura \ref{fig:cobertura_fuentes} muestra que Steam cubre 80.1\% de los participantes, biweekly 62.7\%, cognici\'on 57.1\% y uso del tiempo 44.6\%. En contraste, Xbox cubre 10.9\%, iOS 3.7\% y Android 2.9\%. El profiling confirma el mismo patr\'on al mostrar faltantes por columna y tipos de dato. La vista de filtros permite restringir el an\'alisis a participantes con una fuente espec\'ifica y comprobar cu\'antos casos quedan antes de interpretar una separaci\'on o un cluster.

\textbf{Interpretaci\'on.} Este caso muestra que el primer conocimiento extra\'ido no es un cluster, sino una advertencia metodol\'ogica: la herramienta permite distinguir entre patrones de comportamiento y patrones producidos por disponibilidad desigual de datos.

\textbf{Validaci\'on de la herramienta.} La propuesta se valida porque incorpora control de calidad visual antes del modelado. Frente a un an\'alisis exploratorio b\'asico, el sistema no solo calcula gr\'aficos, sino que orienta al usuario sobre qu\'e evidencia es m\'as fuerte y qu\'e evidencia requiere cautela. Por tanto, este caso demuestra que la herramienta no solo visualiza datos, sino que permite validar un patr\'on mediante la relaci\'on entre cobertura, variables originales y vistas coordinadas.

\subsection{Validaci\'on de la propuesta visual}

La herramienta permite encontrar conocimiento porque transforma datos heterog\'eneos en una exploraci\'on guiada. No se limita a mostrar distribuciones; permite formular preguntas, seleccionar casos, comparar participantes, revisar cobertura y explicar grupos con variables originales.

Las vistas responden a las tareas anal\'iticas planteadas. El resumen BI responde a la calidad de datos; profiling y vector responden a la selecci\'on de variables; la matriz de correlaci\'on y la vista bivariada responden a relaciones entre variables; la reducci\'on dimensional responde a similitud y separaci\'on visual; el clustering responde a perfiles exploratorios; y la selecci\'on coordinada responde a validaci\'on de casos.

Los patrones se confirman con m\'as de una visualizaci\'on. Por ejemplo, la Figura \ref{fig:pca_clusters} permite ubicar perfiles en la proyecci\'on 2D; el perfil de juego nocturno se interpreta con diferencias estandarizadas en la Figura \ref{fig:perfil_clusters} y se contrasta en la vista bivariada de la Figura \ref{fig:intensidad_nocturna}; el perfil de mayor malestar se valida con la Figura \ref{fig:bienestar_riesgo}; y el perfil multiplataforma se valida con cobertura por cluster en la Figura \ref{fig:cobertura_cluster}. Esta triangulaci\'on reduce el riesgo de sacar conclusiones por una sola vista.

La interacci\'on entre vistas ayuda a profundizar el an\'alisis porque el usuario puede pasar de una pregunta general a un caso concreto. Frente a un an\'alisis exploratorio b\'asico, la propuesta aporta trazabilidad del razonamiento: se puede explicar qu\'e variables se usaron, qu\'e modelo produjo la proyecci\'on, qu\'e grupo se detect\'o, qu\'e variables lo diferencian y qu\'e limitaciones tiene la interpretaci\'on.

\section{Conclusiones}

El proyecto logr\'o construir y validar una herramienta de visualizaci\'on anal\'itica para explorar el dataset OpenPlay desde una perspectiva multivariada. OpenPlay Profiler integra bienestar, riesgo de juego problem\'atico, telemetr\'ia, cognici\'on, plataformas y uso del tiempo en un flujo que va desde la revisi\'on de calidad de datos hasta la interpretaci\'on de perfiles exploratorios. La contribuci\'on principal no es solo mostrar una proyecci\'on de participantes, sino permitir que cada patr\'on observado se contraste con variables originales, cobertura y vistas de confirmaci\'on.

Las tareas anal\'iticas cubiertas incluyen evaluar cobertura, seleccionar y justificar el vector de caracter\'isticas, revisar relaciones entre variables, proyectar participantes, aplicar clustering, interpretar subgrupos y validar hallazgos mediante estudios de caso. Estas tareas se conectan con vistas espec\'ificas de la interfaz: datos y profiling para calidad, vector para selecci\'on de atributos, bivariado y encuestas para confirmaci\'on, reducci\'on dimensional para exploraci\'on de grupos, filtros para control de submuestras y validaci\'on para sintetizar la evidencia.

Los estudios de caso permitieron validar cuatro hallazgos principales. Primero, la cobertura de fuentes es desigual: Steam, biweekly y cognici\'on ofrecen mayor evidencia, mientras que Android, iOS y Xbox deben interpretarse con cautela. Segundo, existe un perfil de alta intensidad y juego nocturno, confirmado por la relaci\'on entre sesiones totales y sesiones nocturnas. Tercero, se identific\'o un perfil con mayor PROMIS, mayor GDT/BANGS, menor WEMWBS y m\'as registros de uso del tiempo relacionados con videojuegos. Cuarto, se observ\'o un perfil de alta cobertura multiplataforma, explicado por mayor n\'umero de plataformas y mayor disponibilidad de telemetr\'ia. Estos perfiles no son diagn\'osticos ni categor\'ias definitivas, sino agrupamientos exploratorios para formular hip\'otesis.

El dise\~no visual ayud\'o a extraer conocimiento porque oblig\'o a validar los patrones desde m\'as de una vista. La proyecci\'on 2D permiti\'o ubicar grupos; el perfil estandarizado de clusters explic\'o qu\'e variables diferenciaban a cada grupo; los gr\'aficos bivariados validaron relaciones espec\'ificas como intensidad y nocturnidad; las vistas de encuestas ayudaron a interpretar bienestar y riesgo; y la cobertura por fuente permiti\'o distinguir entre comportamiento observado y disponibilidad desigual de datos. Esta coordinaci\'on reduce el riesgo de concluir solo por la forma de una nube de puntos.

Entre las limitaciones se encuentran la alta presencia de valores faltantes, la cobertura desigual por plataforma, la dependencia del vector seleccionado, el uso de imputaci\'on para completar variables y la sensibilidad de PCA, UMAP, t-SNE y clustering a sus par\'ametros. Como mejoras futuras se propone evaluar estabilidad de clusters con varias configuraciones, comparar clustering en el espacio original estandarizado y en proyecciones 2D, incorporar m\'etricas estad\'isticas de separaci\'on entre perfiles, enriquecer el diccionario de datos y generar reportes autom\'aticos de los estudios de caso desde la propia interfaz.
